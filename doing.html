FileZilla 사용법 완벽 마스터

FTP 클라이언트 FileZilla의 기초부터 고급 활용까지!

FileZilla란 무엇인가?

FileZilla는 FTP (File Transfer Protocol), FTPS (FTP over SSL/TLS), SFTP (SSH File Transfer Protocol)를 지원하는 매우 인기있는 무료 오픈소스 FTP 클라이언트입니다. 주된 목적은 사용자의 로컬 컴퓨터와 웹 서버 간에 파일을 쉽고 안전하게 주고받는 것입니다. 웹사이트 개발자, 시스템 관리자뿐만 아니라 개인 사용자들도 파일 관리를 위해 FileZilla를 널리 사용합니다.

왜 FileZilla를 사용해야 할까요?

  • 완전 무료 및 오픈소스: 어떠한 비용 부담 없이 모든 강력한 기능을 자유롭게 사용할 수 있습니다.
  • 광범위한 프로토콜 지원: FTP, FTPS, SFTP 등 주요 파일 전송 프로토콜을 모두 지원하여, 거의 모든 종류의 서버 환경에 유연하게 접속할 수 있습니다.
  • 직관적이고 사용하기 쉬운 인터페이스: 드래그 앤 드롭 기능을 포함하여, 초보자도 몇 번의 클릭만으로 파일 전송을 마스터할 수 있도록 설계되었습니다.
  • 강력하고 다양한 기능: 사이트 관리자, 빠른 연결, 디렉터리 비교, 전송 큐 관리, 파일 검색, 권한 변경(CHMOD) 등 효율적인 파일 관리를 위한 전문가 수준의 기능을 제공합니다.
  • 크로스 플랫폼 지원: Windows, macOS, Linux 등 주요 데스크톱 운영체제에서 완벽하게 작동하여, 어떤 환경에서든 동일한 사용자 경험을 제공합니다.
  • 안정적인 대용량 파일 전송: 대용량 파일도 끊김 없이 안정적으로 전송할 수 있으며, 중단된 전송을 이어받는 기능도 지원합니다.

FileZilla 설치하기

FileZilla를 사용하기 위한 첫 단계는 프로그램을 다운로드하고 설치하는 것입니다. 아래 단계를 따라 진행하세요.

따라하기:

  1. FileZilla 공식 웹사이트 (filezilla-project.org)에 방문합니다.
  2. 메인 페이지에서 "Download FileZilla Client" 버튼을 클릭합니다. (주의: "FileZilla Server"가 아닌 "FileZilla Client"를 다운로드해야 합니다.)
    [FileZilla 공식 웹사이트의 'Download FileZilla Client' 버튼 위치 스크린샷]
  3. 사용 중인 운영체제(Windows, macOS, Linux)에 맞는 버전을 선택하여 다운로드합니다. 대부분의 경우 웹사이트가 자동으로 운영체제를 감지하여 적절한 버전을 추천해줍니다.
  4. 다운로드한 설치 파일 (예: FileZilla_x.xx.x_win64-setup.exe 또는 FileZilla_x.xx.x_macosx-x86.app.tar.bz2)을 실행합니다.
  5. 설치 마법사가 나타나면 화면의 지시에 따라 설치를 진행합니다.
    • 라이선스 계약에 동의 (I Agree).
    • 설치 옵션 선택 (일반적으로 기본값을 유지하는 것이 좋습니다. 다만, 번들 소프트웨어 제안이 있을 경우 주의하여 선택 해제하세요).
    • 설치 경로 지정 (기본 경로 권장).
    • 구성 요소 선택 (기본값 유지).
    • 시작 메뉴 폴더 지정.
    [FileZilla 설치 마법사의 단계별 화면 예시 (라이선스 동의, 설치 옵션 등)]
  6. 설치가 완료되면 "Finish" 버튼을 클릭하여 마법사를 종료합니다. FileZilla가 자동으로 실행되도록 설정할 수도 있습니다.

FileZilla 핵심 기본 기능 마스터하기

FileZilla의 인터페이스와 기본적인 파일/폴더 조작 방법을 익혀보세요. 이것만 알아도 파일 관리가 훨씬 쉬워집니다.

1. FileZilla 인터페이스 살펴보기

[FileZilla 전체 인터페이스 스크린샷과 각 주요 영역(빠른 연결 바, 메시지 로그, 로컬 사이트, 리모트 사이트, 전송 큐) 표시]
  • 빠른 연결 바: 임시로 서버에 빠르게 접속할 때 사용합니다. (호스트, 사용자명, 비밀번호, 포트 입력)
  • 메시지 로그 창: 서버와의 통신 내용, 명령어, 오류 메시지 등이 표시됩니다. 문제 해결에 유용합니다.
  • 로컬 사이트 창 (왼쪽): 사용자의 컴퓨터에 있는 파일과 폴더를 보여줍니다.
  • 리모트 사이트 창 (오른쪽): 접속한 FTP 서버에 있는 파일과 폴더를 보여줍니다.
  • 전송 큐 창 (하단): 파일 전송 대기 목록, 성공/실패한 전송 내역을 보여줍니다.

2. 서버 접속 및 해제

  • 접속: '사이트 관리자'를 이용하거나 '빠른 연결 바'에 정보를 입력 후 '빠른 연결' 버튼 클릭.
  • 해제: 상단 툴바의 빨간색 X 모양 아이콘 ('현재 서버와의 연결 끊기') 클릭, 또는 메뉴에서 '서버(S)' > '연결 끊기(D)' 선택.
    [FileZilla 연결 해제 버튼 아이콘 스크린샷]

3. 파일 및 폴더 기본 조작 (로컬/리모트 공통)

로컬 사이트 창이나 리모트 사이트 창에서 파일/폴더를 선택하고 마우스 오른쪽 버튼을 클릭하면 다양한 작업을 할 수 있습니다.

[파일/폴더 우클릭 시 나타나는 컨텍스트 메뉴 예시 (새 디렉터리 만들기, 삭제, 이름 바꾸기 등)]
  1. 새 디렉터리(폴더) 만들기: 빈 공간에 우클릭 후 '디렉터리 만들기(C)...' 선택, 이름 입력 후 확인.
  2. 이름 바꾸기: 파일/폴더 선택 후 우클릭 > '이름 바꾸기(R)' 선택 (또는 F2 키), 새 이름 입력 후 엔터.
  3. 삭제하기: 파일/폴더 선택 후 우클릭 > '삭제(D)' 선택 (또는 Delete 키). 삭제 전 확인 창이 나타납니다. (주의: 서버에서 삭제된 파일은 복구가 어려울 수 있습니다!)
  4. 새로고침: 파일 목록이 최신 상태로 보이지 않을 때 우클릭 > '새로고침(F)' 선택 (또는 F5 키).

사이트 관리자 사용법

사이트 관리자는 자주 접속하는 FTP 서버의 정보를 저장하고 쉽게 접속할 수 있도록 도와주는 매우 유용한 기능입니다. 매번 접속 정보를 입력할 필요가 없어 편리합니다.

따라하기:

  1. FileZilla를 실행합니다.
  2. 상단 메뉴에서 '파일(F)' > '사이트 관리자(S)...'를 클릭하거나, 키보드 단축키 Ctrl+S (macOS: Cmd+S)를 누릅니다.
    [FileZilla 상단 메뉴에서 '사이트 관리자' 선택하는 스크린샷]
  3. '사이트 관리자' 창이 나타나면 왼쪽 하단의 '새 사이트(N)' 버튼을 클릭합니다.
  4. '내 사이트' 목록 아래에 "새 사이트"라는 이름으로 항목이 생성됩니다. 이 이름을 식별하기 쉬운 이름으로 변경합니다 (예: "내 웹호스팅", "회사 FTP 서버").
  5. 오른쪽 '일반' 탭에서 서버 접속 정보를 정확히 입력합니다:
    [FileZilla 사이트 관리자 창의 '일반' 탭 상세 입력 항목 스크린샷 (프로토콜, 호스트, 포트, 로그온 유형, 사용자, 비밀번호 등)]
    • 프로토콜(T): 서버에서 지원하는 프로토콜을 선택합니다 (일반적으로 FTP 또는 SFTP - SSH File Transfer Protocol). 잘 모를 경우 웹호스팅 업체나 서버 관리자에게 문의하세요.
    • 호스트(H): FTP 서버의 주소를 입력합니다 (예: ftp.example.com 또는 IP 주소 123.45.67.89).
    • 포트(P): 프로토콜에 따른 기본 포트 번호가 자동으로 입력됩니다 (FTP: 21, SFTP: 22). 서버에서 별도의 포트를 사용하는 경우 해당 포트 번호를 입력합니다.
    • 암호화(E): FTPS (FTP over SSL/TLS)를 사용하는 경우 선택합니다. (예: 'Require explicit FTP over TLS' 또는 'Require implicit FTP over TLS'). SFTP는 자체적으로 암호화됩니다.
    • 로그온 유형(L): 접속 방식을 선택합니다.
      • 일반(N): 사용자 이름과 비밀번호를 저장해두고 자동으로 로그인합니다. (개인 PC에서만 사용 권장)
      • 비밀번호 묻기(A): 접속 시마다 비밀번호를 입력하도록 합니다. (보안상 권장)
      • 계정(C): (드물게 사용)
      • 키 파일(K): SFTP 접속 시 비밀번호 대신 개인 키 파일을 사용하여 인증합니다. (고급 사용자용)
    • 사용자(U): FTP 계정 아이디(사용자명)를 입력합니다.
    • 비밀번호(W): (로그온 유형이 '일반'인 경우) FTP 계정 비밀번호를 입력합니다. '비밀번호 묻기'로 설정했다면 비워두거나 임의의 값을 입력해도 됩니다.
  6. (선택 사항) '고급' 탭에서 기본 로컬/리모트 디렉터리 지정, '전송 설정' 탭에서 문자셋(인코딩) 설정 (UTF-8 권장) 등을 할 수 있습니다.
  7. 정보 입력이 완료되면 '연결(C)' 버튼을 클릭하여 바로 서버에 접속하거나, '확인(O)' 버튼을 클릭하여 정보를 저장하고 창을 닫습니다.
  8. 저장된 사이트는 다음부터 사이트 관리자에서 해당 사이트를 선택 후 '연결'만 누르면 간편하게 접속할 수 있습니다.

빠른 연결 사용법

빠른 연결은 사이트 관리자에 서버 정보를 저장하지 않고 일회성으로 FTP 서버에 접속할 때 사용합니다. 하지만 접속 정보가 저장되지 않으므로, 자주 사용하는 서버는 사이트 관리자에 등록하는 것이 훨씬 효율적입니다.

따라하기:

  1. FileZilla를 실행합니다.
  2. 화면 상단 툴바 아래에 있는 '빠른 연결' 바에 접속 정보를 입력합니다.
    [FileZilla 빠른 연결 바 (호스트, 사용자명, 비밀번호, 포트 입력 칸) 스크린샷]
    • 호스트: 접속할 FTP 서버 주소 (예: sftp.example.com). SFTP 접속 시에는 주소 앞에 sftp:// 를 붙여주면 명시적으로 SFTP 프로토콜을 사용합니다 (예: sftp://example.com).
    • 사용자명: FTP 계정 아이디.
    • 비밀번호: FTP 계정 비밀번호.
    • 포트: (선택 사항) 특정 포트 번호를 사용한다면 입력합니다. 비워두면 프로토콜 기본 포트(FTP: 21, SFTP: 22)를 사용합니다.
  3. 정보를 모두 입력한 후 오른쪽에 있는 '빠른 연결(Q)' 버튼을 클릭하거나 키보드에서 Enter 키를 누릅니다.
  4. 서버에 성공적으로 연결되면 로컬 사이트와 리모트 사이트 창에 파일 목록이 표시되고, 메시지 로그 창에 접속 성공 메시지가 나타납니다.
  5. 주의: 빠른 연결로 접속한 정보는 FileZilla를 종료하면 사라집니다. 또한, 비밀번호가 화면에 잠시 보일 수 있으므로 공용 PC에서는 사용에 주의해야 합니다. 최근 접속 기록은 빠른 연결 바의 드롭다운 화살표를 클릭하여 확인할 수 있습니다.

파일 업로드 및 다운로드

FileZilla의 핵심 기능은 로컬 컴퓨터와 원격 서버 간에 파일을 주고받는 것입니다. 인터페이스는 일반적으로 왼쪽이 로컬(내 컴퓨터), 오른쪽이 서버(리모트)를 나타냅니다.

[FileZilla 메인 화면: 로컬(왼쪽 패널)과 서버(오른쪽 패널)의 파일/폴더 목록이 보이는 스크린샷]

파일 업로드 (로컬 컴퓨터 → 서버)

내 컴퓨터에 있는 파일을 웹 서버로 올리는 과정입니다.

  1. FileZilla를 통해 서버에 성공적으로 접속합니다.
  2. 왼쪽 '로컬 사이트' 창에서 업로드할 파일이나 폴더가 있는 경로로 이동합니다. (예: D:\MyWebsite\images)
  3. 오른쪽 '리모트 사이트' 창에서 파일이나 폴더를 업로드할 서버 내의 목적지 경로로 이동합니다. (예: /public_html/images 또는 /www/data)
  4. 파일/폴더 업로드 방법 (다음 중 가장 편한 방법 선택):
    • 드래그 앤 드롭: 로컬 사이트 창에서 원하는 파일/폴더를 마우스로 클릭한 채로 리모트 사이트 창의 목적지 경로로 끌어다 놓습니다. 가장 직관적인 방법입니다.
      [드래그 앤 드롭으로 로컬 파일을 서버로 업로드하는 동작 GIF 또는 스크린샷]
    • 더블 클릭: 로컬 사이트 창에서 업로드할 파일/폴더를 더블 클릭합니다. (설정에 따라 다를 수 있으며, 기본적으로는 현재 열린 리모트 디렉터리로 업로드됩니다.)
    • 마우스 오른쪽 클릭 메뉴: 로컬 사이트 창에서 업로드할 파일/폴더를 선택하고 마우스 오른쪽 버튼을 클릭한 후, 나타나는 컨텍스트 메뉴에서 '업로드(U)'를 선택합니다.
  5. 파일 전송이 시작되면 화면 하단의 '전송 큐' 탭에서 진행 상황(남은 파일, 전송 속도 등)을 실시간으로 확인할 수 있습니다.

파일 다운로드 (서버 → 로컬 컴퓨터)

웹 서버에 있는 파일을 내 컴퓨터로 내려받는 과정입니다.

  1. FileZilla를 통해 서버에 성공적으로 접속합니다.
  2. 오른쪽 '리모트 사이트' 창에서 다운로드할 파일이나 폴더가 있는 경로로 이동합니다.
  3. 왼쪽 '로컬 사이트' 창에서 파일이나 폴더를 다운로드하여 저장할 로컬 컴퓨터의 목적지 경로로 이동합니다.
  4. 파일/폴더 다운로드 방법 (업로드와 유사):
    • 드래그 앤 드롭: 리모트 사이트 창에서 원하는 파일/폴더를 로컬 사이트 창의 목적지 경로로 끌어다 놓습니다.
    • 더블 클릭: 리모트 사이트 창에서 다운로드할 파일/폴더를 더블 클릭합니다.
    • 마우스 오른쪽 클릭 메뉴: 리모트 사이트 창에서 다운로드할 파일/폴더를 선택하고 마우스 오른쪽 버튼을 클릭한 후, '다운로드(D)'를 선택합니다.
  5. 마찬가지로 '전송 큐'에서 진행 상황을 확인할 수 있습니다.

팁: 여러 파일이나 폴더를 한 번에 선택하여 (키보드에서 Ctrl 키를 누른 채 클릭하거나, Shift 키를 누른 채 범위 선택) 업로드하거나 다운로드할 수 있습니다. 폴더를 전송하면 하위 파일과 폴더까지 모두 함께 전송됩니다.

웹 페이지 테스트 및 실행 (오해와 진실)

많은 분들이 FileZilla로 HTML 파일을 "실행"하거나 "테스트"할 수 있다고 생각하지만, 이는 정확한 이해가 아닙니다. 이 섹션에서 FileZilla의 역할과 올바른 웹 페이지 테스트 방법을 명확히 알려드립니다.

중요: FileZilla는 웹 서버나 웹 브라우저가 아닙니다!

FileZilla는 파일을 로컬 컴퓨터와 웹 서버 간에 전송하는 도구(FTP 클라이언트)입니다. FileZilla 자체에는 HTML 코드를 해석하여 웹 페이지처럼 보여주거나, PHP 같은 서버사이드 스크립트를 실행하는 기능이 없습니다.

올바른 웹 페이지 테스트 절차

웹 페이지 (예: index.html) 또는 웹 애플리케이션을 테스트하려면 다음 단계를 따라야 합니다:

  1. 웹 파일 준비:
    • 로컬 컴퓨터에서 index.html, CSS 파일 (style.css), JavaScript 파일 (script.js), 이미지 파일 등을 만듭니다.
    • 예시 index.html 파일: 
      <!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <title>나의 첫 웹 페이지</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <h1>안녕하세요, FileZilla로 업로드한 페이지입니다!</h1>
    <p>이 페이지는 성공적으로 웹 서버에 올라갔습니다.</p>
    <script src="script.js"></script>
</body>
</html>
  2. FileZilla로 서버에 업로드:
    • FileZilla를 사용하여 웹 서버에 접속합니다.
    • 웹 서버의 지정된 폴더 (보통 public_html, www, htdocs, httpdocs 등) 안에 준비한 웹 파일들을 업로드합니다. 이 폴더가 웹에서 접근 가능한 루트 디렉터리입니다.
      [FileZilla로 public_html 폴더에 index.html 파일을 업로드하는 스크린샷]
    • 만약 하위 폴더(예: mytest)를 만들어 그 안에 업로드했다면, 해당 경로를 기억해야 합니다.
  3. 웹 브라우저로 확인:
    • 크롬(Chrome), 엣지(Edge), 파이어폭스(Firefox) 등의 웹 브라우저를 엽니다.
    • 주소창에 웹 서버의 주소(도메인 또는 IP)와 파일 경로를 입력하여 접속합니다.
      • 루트 디렉터리에 index.html을 업로드한 경우: http://내도메인.com 또는 http://서버IP주소
      • mytest 폴더 안에 index.html을 업로드한 경우: http://내도메인.com/mytest/ 또는 http://내도메인.com/mytest/index.html
      [웹 브라우저 주소창에 URL을 입력하여 접속한 화면 예시]
    • 업로드한 HTML 페이지가 웹 브라우저에 정상적으로 표시되는지 확인합니다. CSS 스타일과 JavaScript 기능도 함께 테스트합니다.

다른 자료 (예: PHP, Python 스크립트) 테스트

.php, .py (웹 프레임워크 사용 시) 같은 서버사이드 스크립트 파일들은 웹 서버에 해당 언어의 실행 환경(인터프리터, 엔진)이 설치되어 있어야 합니다. FileZilla로 파일을 업로드한 후, 웹 브라우저를 통해 해당 파일의 URL로 접속하면 서버가 스크립트를 실행하고 그 결과를 HTML 형태로 브라우저에 보내줍니다.

예를 들어, info.php 파일을 서버에 업로드했다면 http://내도메인.com/info.php로 접속하여 실행 결과를 확인합니다.

결론: FileZilla는 "문지기"처럼 파일을 서버로 안전하게 옮겨주는 역할을 합니다. 실제 웹 페이지가 "살아서 움직이는" 것은 웹 서버와 웹 브라우저 덕분입니다!

FileZilla 고급 활용 팁

FileZilla의 기본 기능을 넘어, 더욱 전문적이고 효율적인 파일 관리를 위한 고급 기능들을 알아보세요.

1. 전송 설정 최적화

네트워크 환경이나 서버 특성에 맞게 전송 설정을 조절하여 안정성과 속도를 개선할 수 있습니다.

  1. 메뉴에서 '편집(E)' > '설정(S)...'을 선택합니다.
  2. 설정 창 왼쪽 트리에서 '전송' 항목을 선택합니다.
    [FileZilla 설정 창의 '전송' 항목 스크린샷 (최대 동시 전송 수, 타임아웃 설정 등)]
  3. 최대 동시 전송 수: 한 번에 몇 개의 파일을 동시에 전송할지 결정합니다. 너무 많으면 서버에 부하를 주거나 불안정해질 수 있고, 너무 적으면 대량 파일 전송 시 오래 걸립니다. (기본값: 2, 권장: 2~5)
  4. 동시 다운로드/업로드 제한: 특정 서버에 대한 동시 접속 수를 제한할 수 있습니다.
  5. 타임아웃: 서버 응답이 없을 때 얼마 동안 기다릴지 설정합니다. 네트워크가 불안정할 경우 값을 늘려볼 수 있습니다. (기본값: 20초)
  6. 전송 속도 제한: FileZilla가 사용하는 인터넷 대역폭을 제한하여 다른 작업에 영향을 덜 주도록 할 수 있습니다. (설정 > '전송' > '속도 제한')

2. 북마크 기능 활용

자주 방문하는 로컬 또는 리모트 디렉터리 경로를 북마크에 추가해두면 클릭 한 번으로 빠르게 이동할 수 있어 매우 편리합니다.

  1. 로컬 또는 리모트 사이트 창에서 원하는 디렉터리로 이동합니다.
  2. 메뉴에서 '북마크(B)' > '북마크 추가(A)...'를 선택하거나, 단축키 Ctrl+B (macOS: Cmd+B)를 누릅니다.
    [FileZilla 북마크 추가 창 스크린샷 (로컬/리모트 경로, 이름 지정)]
  3. 북마크 이름을 지정하고, '로컬 사이트' 또는 '리모트 사이트' 중 현재 경로를 저장할 위치를 선택합니다. '사이트별 북마크'로 설정하면 현재 접속한 서버에만 적용되는 북마크를 만들 수도 있습니다.
  4. '확인'을 누르면 북마크가 추가됩니다. 추가된 북마크는 '북마크(B)' 메뉴에서 선택하여 바로 이동할 수 있습니다.

3. 파일 검색 (로컬 및 리모트)

특정 파일을 찾기 어려울 때 파일 검색 기능을 사용하면 유용합니다.

  • 리모트 파일 검색: (이미 '기타 유용한 기능들'에 설명됨) 메뉴 '서버(S)' > '리모트 파일 검색(F)...'
  • 로컬 파일 검색: FileZilla 자체에는 강력한 로컬 파일 검색 기능이 내장되어 있지는 않지만, 로컬 사이트 창에서 경로를 탐색하며 파일을 찾을 수 있습니다. 복잡한 검색은 운영체제의 파일 탐색기(Windows 탐색기, macOS Finder)를 이용하는 것이 더 효율적입니다.

4. 로그 기록 확인 및 활용

FileZilla 창 상단에 있는 메시지 로그 창은 서버와의 모든 통신 기록을 보여줍니다. 접속 오류, 전송 실패 등의 문제가 발생했을 때 이 로그를 자세히 살펴보면 원인을 파악하는 데 큰 도움이 됩니다.

[FileZilla 메시지 로그 창 스크린샷 (오류 메시지 또는 성공적인 통신 기록 예시)]
  • 로그 수준은 '편집(E)' > '설정(S)...' > '로깅'에서 조절할 수 있습니다. (문제 해결 시 '자세히'로 설정)
  • 로그 내용을 파일로 저장하여 나중에 분석하거나 지원팀에 전달할 수도 있습니다.

이 외에도 '사이트 관리자'의 '고급' 탭에서 서버 유형(Server Type)을 특정하거나, '문자셋'을 'UTF-8로 강제'하는 등의 설정은 특정 서버 환경에서 발생할 수 있는 호환성 문제를 해결하는 데 도움이 될 수 있습니다.

디렉터리 비교

디렉터리 비교 기능은 로컬 컴퓨터와 원격 서버에 있는 특정 폴더의 내용을 비교하여 어떤 파일이 다르거나, 한쪽에만 존재하는지 등을 시각적으로 보여줍니다. 웹사이트 업데이트 시 누락된 파일을 확인하거나 동기화 상태를 점검할 때 매우 유용합니다.

따라하기:

  1. FileZilla를 통해 서버에 접속합니다.
  2. 로컬 사이트 창과 리모트 사이트 창에서 비교하고자 하는 동일한 경로 또는 대응되는 경로로 각각 이동합니다. (예: 로컬의 C:\MyWebSite\projectA 와 서버의 /public_html/projectA)
  3. 상단 메뉴에서 '보기(V)' > '디렉터리 비교(C)' > '활성화(E)'를 선택합니다. (단축키: Ctrl+O 또는 macOS: Cmd+O). 또는 툴바의 디렉터리 비교 아이콘을 클릭할 수도 있습니다.
    [FileZilla 상단 메뉴에서 '디렉터리 비교' > '활성화' 선택하는 스크린샷 또는 툴바 아이콘]
  4. 비교 기준을 선택할 수 있습니다. 기본적으로 '파일 크기 또는 수정 시간'으로 설정되어 있습니다. 필요시 '보기(V)' > '디렉터리 비교(C)' 메뉴 하위에서 '파일 크기만 비교(S)' 또는 '수정 시간만 비교(M)'를 선택하여 변경할 수 있습니다. '숨김 파일 비교(H)' 옵션도 유용합니다.
  5. 비교가 활성화되면 파일 목록의 아이콘이나 배경색이 변경되어 차이점을 명확하게 표시합니다:
    [FileZilla 디렉터리 비교 결과: 파일 목록에 노란색, 녹색, 빨간색 등으로 차이가 표시된 스크린샷]
    • 노란색 강조: 해당 파일이 로컬 디렉터리에만 존재합니다 (서버에는 없음).
    • 녹색 강조: 로컬 디렉터리의 파일이 서버의 파일보다 최신 버전입니다 (업로드 필요 가능성).
    • 빨간색 강조: 서버 디렉터리의 파일이 로컬 파일보다 최신이거나, 서버에만 파일이 존재합니다 (다운로드 필요 또는 로컬에 없는 파일).
    • 회색 또는 기본색 (강조 없음): 양쪽 디렉터리의 파일이 (선택한 기준에 따라) 동일하거나, 비교 대상이 아닙니다. (정확한 색상 표시는 FileZilla 버전 및 테마에 따라 약간 다를 수 있습니다.)
  6. 차이점을 확인한 후, 필요한 파일을 선택하여 업로드하거나 다운로드하여 양쪽 디렉터리를 동기화할 수 있습니다.
  7. 디렉터리 비교 기능을 끄려면 다시 '보기(V)' > '디렉터리 비교(C)' > '비활성화(D)'를 선택하거나 툴바 아이콘을 다시 클릭합니다.

전송 큐 관리

전송 큐는 FileZilla 창 하단에 위치하며, 현재 진행 중이거나 대기 중인 파일 전송 목록을 보여줍니다. 또한 성공한 전송과 실패한 전송 내역도 이곳에서 확인할 수 있어, 파일 전송 과정을 체계적으로 관리할 수 있습니다.

[FileZilla 창 하단의 전송 큐 영역 (대기 파일, 실패한 전송, 성공한 전송 탭) 스크린샷]

주요 탭 및 사용법:

  • 대기 파일 탭 (Queued files):
    • 업로드 또는 다운로드를 시작하면 파일들이 이 탭에 순서대로 추가됩니다.
    • 파일 목록, 크기, 우선순위, 상태 (대기 중, 전송 중, 남은 시간 등)를 한눈에 확인할 수 있습니다.
    • 특정 파일을 마우스 오른쪽 버튼으로 클릭하여 다음과 같은 작업을 수행할 수 있습니다:
      [대기 파일 탭에서 파일 우클릭 시 나타나는 컨텍스트 메뉴 스크린샷 (처리 중지, 우선순위 변경 등)]
      • 처리 중지 및 제거: 선택한 파일의 전송을 중단하고 큐에서 완전히 제거합니다.
      • 우선순위 변경: 전송 순서를 조정합니다 (가장 높음, 높음, 보통, 낮음, 가장 낮음). 중요한 파일을 먼저 보낼 때 유용합니다.
      • 선택한 파일만 처리: 다른 파일은 잠시 멈추고 선택한 파일(들)부터 전송합니다.
      • 큐 처리 시작/중지: 전체 전송 큐의 작동을 일시 중지하거나 다시 시작합니다. (툴바의 재생/일시정지 아이콘으로도 제어 가능)
  • 실패한 전송 탭 (Failed transfers):
    • 네트워크 오류, 권한 문제, 디스크 공간 부족 등으로 전송에 실패한 파일들이 이 탭으로 이동합니다.
    • 실패 원인은 메시지 로그 창에서 더 자세히 확인할 수 있습니다.
    • 실패한 파일을 마우스 오른쪽 버튼으로 클릭하여 '모든 항목 초기화 및 재전송(R)' 또는 '선택 항목 초기화 및 재전송'을 통해 다시 시도할 수 있습니다. '큐에서 제거'도 가능합니다.
  • 성공한 전송 탭 (Successful transfers):
    • 성공적으로 전송이 완료된 파일들의 목록을 보여줍니다.
    • 전송된 파일, 크기, 걸린 시간 등의 정보를 확인할 수 있습니다.
    • 목록을 지우려면 마우스 오른쪽 클릭 후 '모든 항목 제거' 또는 '선택 항목 제거'를 선택합니다. (실제 파일이 삭제되는 것은 아닙니다.)
  • 동시에 처리할 수 있는 전송 개수는 '편집(E)' > '설정(S)...' > '전송' 메뉴에서 '최대 동시 전송 수'를 조절하여 변경할 수 있습니다.

기타 유용한 기능들

FileZilla에는 기본적인 파일 전송 외에도 작업 효율을 높여주는 다양한 부가 기능들이 있습니다. 이미 다른 섹션에서 언급된 기능도 있지만, 여기서 다시 한번 정리합니다.

1. 파일/폴더 권한 변경 (CHMOD)

서버에 업로드된 파일이나 폴더의 접근 권한(읽기, 쓰기, 실행)을 변경하는 기능입니다. 웹 서버에서 PHP 스크립트 실행 권한(예: 755)이나 특정 파일의 쓰기 권한(예: 666 또는 644) 등을 설정할 때 반드시 필요합니다.

  1. 리모트 사이트 창에서 권한을 변경할 파일이나 폴더를 선택합니다.
  2. 마우스 오른쪽 버튼을 클릭하고 컨텍스트 메뉴에서 '파일 권한(F)...'을 선택합니다.
  3. '파일 속성 변경' 창이 나타나면, '숫자 값(N)'에 직접 권한 값(예: 755, 644)을 입력하거나, 아래의 체크박스(읽기, 쓰기, 실행 권한을 소유자/그룹/공용으로 구분)를 이용하여 원하는 권한을 설정합니다.
    [FileZilla 파일 권한 변경(CHMOD) 창 스크린샷 (숫자 값 입력 및 체크박스)]
  4. '하위 디렉터리로의 재귀(R)' 옵션을 선택하면 선택한 폴더 내의 모든 파일과 하위 폴더에도 동일한 권한을 일괄적으로 적용할 수 있습니다. (매우 주의해서 사용해야 합니다!)
  5. '확인'을 클릭하여 변경 사항을 서버에 적용합니다.

2. 동기화 브라우징

로컬 디렉터리와 리모트 디렉터리의 구조가 유사할 때, 한쪽 창에서 디렉터리를 이동하면 다른 쪽 창도 자동으로 해당 디렉터리(또는 이름이 같은 디렉터리)로 함께 이동하는 기능입니다. 양쪽의 동일한 위치를 계속 보면서 작업할 때 매우 편리하여 작업 실수를 줄일 수 있습니다.

  1. 상단 메뉴에서 '보기(V)' > '동기화 브라우징(Y)'을 선택하여 활성화/비활성화합니다. 또는 툴바에 있는 해당 아이콘(양쪽 화살표 모양)을 클릭하여 켜고 끌 수 있습니다.
    [FileZilla 동기화 브라우징 툴바 아이콘 또는 메뉴 위치 스크린샷]
  2. 활성화되면, 로컬 또는 리모트 창에서 디렉터리를 변경할 때 다른 쪽 창도 가능한 경우 동일한 이름의 하위 디렉터리로 이동하려고 시도합니다. 만약 대응하는 디렉터리가 없으면 이동하지 않습니다.

3. 파일명 필터

특정 조건에 맞는 파일이나 디렉터리만 표시하거나 숨기는 기능입니다. 예를 들어, 버전 관리 시스템 파일(.git, .svn 폴더), 운영체제 임시 파일(Thumbs.db, .DS_Store) 등을 숨기거나, 특정 확장자(예: .jpg, .php) 파일만 보고 싶을 때 유용합니다.

  1. 상단 메뉴에서 '보기(V)' > '파일명 필터(N)...'를 클릭하거나 단축키 Ctrl+I (macOS: Cmd+I)를 누릅니다.
  2. '디렉터리 목록 필터' 창이 나타납니다. 왼쪽 목록에서 미리 정의된 필터(예: 'CVS와 SVN 파일 제외', '임시 파일 제외')를 선택하여 활성화/비활성화할 수 있습니다.
    [FileZilla 파일명 필터 설정 창 스크린샷 (필터 목록 및 편집 버튼)]
  3. 새로운 필터를 만들거나 기존 필터를 수정하려면 '필터 규칙 편집...' 버튼을 클릭합니다. 여기서 필터 이름, 조건(파일명, 크기, 속성 등), 적용 방식(일치/불일치, 대소문자 구분 등)을 상세하게 설정할 수 있습니다.
  4. 필터는 로컬 및 리모트 사이트 모두에 적용할 수 있습니다.
